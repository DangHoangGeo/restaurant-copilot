# ShopCopilot iOS App

## Description

ShopCopilot is a multifaceted application designed to streamline operations for businesses (tenants, e.g., restaurants) and provide a seamless experience for their customers. This iOS application serves as a mobile front-end for both tenant administration (Owner/Admin role) and customer interaction (Customer role).

The project is built using SwiftUI and follows a basic MVVM-like pattern for organizing views, view models, services (data handling), and models.

## Project Structure

The main source code for the application is located within the `ShopCopilot/ShopCopilot/` directory. Key subdirectories include:

*   **`Views/`**: Contains all SwiftUI views that make up the user interface.
    *   `ContentView.swift`: The main entry view that orchestrates the app's flow (tenant selection, login, role-based navigation).
    *   `LoginView.swift`: Handles user authentication.
    *   `LanguageSelectionView.swift`: Allows users to change the app's language.
    *   `OwnerAdminDashboardView.swift`: Placeholder dashboard for Owner/Admin users.
    *   `CustomerMenuView.swift`: Placeholder menu view for Customer users.
*   **`ViewModels/`**: Contains `ObservableObject` classes that prepare and manage data for the Views.
    *   `TenantViewModel.swift`: Manages tenant selection and current tenant data.
    *   `AuthenticationViewModel.swift`: Manages user authentication state and logic.
*   **`Models/`**: Contains data structures (structs, enums) used throughout the application.
    *   `Tenant.swift`: Defines the structure for tenant data.
    *   `UserRole.swift`: Defines user roles within the app.
*   **`Services/`**: Contains classes responsible for data fetching and business logic. Currently, these services use mock data.
    *   `TenantService.swift`: Simulates fetching tenant data.
    *   `AuthService.swift`: Simulates user authentication.
*   **`Resources/`**: Contains assets and localization files.
    *   `en.lproj/`, `ja.lproj/`, `vi.lproj/`: Store `Localizable.strings` files for English, Japanese, and Vietnamese respectively.
    *   `i18n_testing_notes.txt`: Provides guidance on testing the internationalization features.
*   **`Utils/`**: Intended for utility functions and extensions (currently empty or minimally used).
*   `ShopCopilotApp.swift`: The main entry point of the application (`@main` struct).
*   `Info.plist`: Application configuration file.

The following directories are structured to hold test files, conventionally placed alongside the main application source code at the `mobileapp/ShopCopilot/` level:
*   **`ShopCopilotTests/`**: Contains unit tests (e.g., `SampleUnitTests.swift`).
*   **`ShopCopilotUITests/`**: Contains UI tests (e.g., `SampleUITests.swift`, `ShopCopilotUITestsLaunchTests.swift`).


## Initial Setup Instructions

### Prerequisites:
*   macOS with Xcode (latest stable version recommended).
*   An Apple Developer account is required if you wish to test the app on a physical iOS device. Simulators can be used without an account.

### Crucial Manual Project Setup:

**The `.xcodeproj` file necessary to open and build the project in Xcode was NOT automatically generated by the development process that created these files.** You must perform the following steps manually:

1.  **Open Xcode.**
2.  Choose **"Create a new Xcode project"**.
3.  Select the **iOS** tab, then choose the **"App"** template and click **Next**.
4.  Enter the following project details:
    *   **Product Name:** `ShopCopilot`
    *   **Team:** Select your Apple Developer team (or "None" if only using simulators).
    *   **Organization Identifier:** A reverse domain name string (e.g., `com.yourname`).
    *   **Interface:** `SwiftUI`
    *   **Language:** `Swift`
    *   **Storage:** None (not applicable for this basic template)
    *   **Include Tests:** Check this box. This will prompt Xcode to create the `ShopCopilotTests` and `ShopCopilotUITests` targets and their respective folders. If you forget, you can add targets later, but it's easier to do it here.
5.  Click **Next**.
6.  **IMPORTANT:** In the save dialog, navigate to the **`mobileapp/ShopCopilot/`** directory (i.e., the parent directory of the `ShopCopilot` folder containing `ShopCopilotApp.swift`). Save your new Xcode project (`ShopCopilot.xcodeproj`) directly into this `mobileapp/ShopCopilot/` directory. **Do not create it inside the inner `ShopCopilot` folder.**
7.  After Xcode creates the new project, it will likely have a default folder structure (e.g., a `ShopCopilot` group with a default `ContentView.swift`, `ShopCopilotApp.swift`, etc., and similar default files in the test targets). **Delete these default files and groups from the Xcode project navigator** (select them and press Delete, choosing "Move to Trash" when prompted for all of them).
8.  **Drag and Drop Source Files:** Open Finder and locate the existing `ShopCopilot` sub-folder (the one located at `mobileapp/ShopCopilot/ShopCopilot/` which contains the `ShopCopilotApp.swift`, `Info.plist`, `Views`, `ViewModels`, etc.). Drag this entire `ShopCopilot` folder from Finder into Xcode's project navigator under the main "ShopCopilot" project group.
    *   When the "Choose options for adding these files" dialog appears:
        *   Ensure **"Create groups"** is selected for "Added folders".
        *   Check the box next to your app's target (`ShopCopilot`).
        *   Click **Finish**.
9.  **Drag and Drop Test Files:**
    *   Drag the `SampleUnitTests.swift` file from `mobileapp/ShopCopilot/ShopCopilotTests/` into the `ShopCopilotTests` group in Xcode's project navigator. Ensure it's added to the `ShopCopilotTests` target.
    *   Drag `SampleUITests.swift` and `ShopCopilotUITestsLaunchTests.swift` from `mobileapp/ShopCopilot/ShopCopilotUITests/` into the `ShopCopilotUITests` group in Xcode. Ensure they are added to the `ShopCopilotUITests` target.
10. **Verify Project Settings:**
    *   **Info.plist:** Ensure the `Info.plist` file from the dragged-in `ShopCopilot` source folder is correctly recognized and used by the main app target. You might need to set its path in Build Settings -> Packaging -> Info.plist File.
    *   **Localization:** Check your project's settings (Project -> Info tab -> Localizations). You should see English, Japanese, and Vietnamese listed. If not, add them.
    *   **Module Name:** Ensure the `@testable import ShopCopilot` in `SampleUnitTests.swift` matches your project's module name (it should by default).
    *   **Deployment Target:** Set an appropriate iOS deployment target (e.g., iOS 15 or later).

## Running the App

1.  Once the project is set up correctly in Xcode, select an iOS Simulator (e.g., iPhone 14 Pro) or a connected physical device from the scheme menu at the top of the Xcode window.
2.  Click the "Play" button (or press Cmd+R) to build and run the application.

## Current Features (High-Level)

*   **Tenant Selection:** Users can manually enter a tenant's subdomain (e.g., "resto1", "resto2") to load tenant-specific context.
*   **Mock Authentication:** Basic login functionality for two roles:
    *   Owner/Admin (e.g., username: "admin", password: "password")
    *   Customer (e.g., username: "customer", password: "password")
*   **Internationalization (i18n):**
    *   Support for English (default), Japanese, and Vietnamese.
    *   A language selection screen allows users to switch languages.
    *   UI elements are localized using `.strings` files.
*   **Role-Based UI:**
    *   After login, users are presented with a `TabView` interface tailored to their role.
    *   **Owner/Admin:** Sees a dashboard tab and a placeholder settings tab.
    *   **Customer:** Sees a menu tab and placeholder tabs for orders and profile.
*   **Basic UI Structure:** Placeholder views for most features, ready for further development.

## Testing

The project is structured to support both Unit Tests and UI Tests.

*   **Unit Tests (`ShopCopilotTests/`):**
    *   The `ShopCopilotTests` target is configured for unit testing application logic.
    *   A placeholder file, `SampleUnitTests.swift`, has been added with basic XCTest boilerplate. This demonstrates a simple functional test and a performance test example.
    *   Writing comprehensive unit tests for ViewModels, Services, and other logic components is a future task.

*   **UI Tests (`ShopCopilotUITests/`):**
    *   The `ShopCopilotUITests` target is configured for UI testing the application's user interface flow.
    *   Placeholder files, `SampleUITests.swift` and `ShopCopilotUITestsLaunchTests.swift`, have been added. These include basic boilerplate for launching the app and taking a screenshot.
    *   Developing UI tests to cover user flows, interactions, and UI element states is a future task.

### Running Tests:
Tests can be run directly from Xcode:
1.  Open the Test navigator (Cmd+6).
2.  Click the play button next to a specific test class, test method, or an entire test target.
3.  Alternatively, use the Product menu (Product > Test).
